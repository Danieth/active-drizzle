## Feasibility verdict (research, 2026-07-18)

The RFC is buildable. Three findings from the code that de-risk it:

1. **`permit` fn form already exists** — `permit?: string[] | ((ctx, ctrl) => string[])` in
   `packages/controller/src/metadata.ts:54`, resolved at runtime in `buildPermittedData`
   (`crud-handlers.ts:224`). And in `defaultUpdate` the record is loaded (line 178) **before**
   permit resolution (line 180) — so record-aware permit is a parameter-threading change,
   not a control-flow change.
2. **The extraction machinery is already the right shape** — ts-morph walks model classes
   (`extractor.ts`), validator bodies are lifted verbatim into generated Clients
   (`react-generator.ts:271`), and the dep-inference engine (`validation-deps.ts`) is a
   receiver-parametrizable walker. Extracting `label/help/copy/presentIf/...` from Attr
   config objects is the same trick on ObjectLiteralExpressions. `presentIf: d => ...`
   needs the walker generalized from `this` to an arbitrary root param — small refactor.
3. **Projection filtering already runs** — `controllerProjectionFields()` +
   `depsFitProjection()` gate which validators ship per controller Client
   (`react-generator.ts:241–251`). `expose` slots in as a redefinition of that set, not
   a new mechanism.

**Honest scope note:** RFC M0–M6 in full is multi-week. The weekend cut line is drawn so
that every phase above it ships something usable and tested; everything below is stretch.

## Phase 2 — Form surface: M3 minimal (Sunday morning) (`react`)

### API v2 — LOCKED (2026-07-18): no `f`, no `Present`

Naming law (codegen-enforced): **camelCase = fields** (`loan.amount`); **PascalCase =
components** (`loan.Form`, `loan.Submit` — column named `Form` ⇒ codegen error + alias hint);
**`$` = framework scalars** (`loan.$status`, `loan.$errors`, `loan.$dirty`, `loan.$submit()`,
`loan.$draft`) — because `status` is too common a column to squat on. JSX member expressions
need no capitalization, so the field handle IS the component (callable + `.error`/`.meta`).
`edit` prop: absent = view · `edit` (bool) = editable w/ Attr's default presenter ·
`edit="name"` = named override. `edit` never inferred (wall #2 intact). Two hooks
(`useEditForm`/`useNewForm`), ONE Handle type, identical JSX — mode lives in the hook.

```tsx
<loan.Form>
  <loan.amount edit />
  <loan.targetRate edit="bpsSlider" label="Target (bps)" />
  <loan.status />
  <loan.assets>{asset => (<><asset.name edit /><asset.value edit /><asset.Remove /></>)}</loan.assets>
  <loan.assets.Add defaults={{ value: 0 }}>Add asset</loan.assets.Add>
  <loan.Submit>Save</loan.Submit>
</loan.Form>
```

Presenter contract (the plugin layer): registry = declaration-merged interface (module
augmentation = how UI-kit packages plug in) + runtime `registerPresenter(name, { kind,
requires?, component })`. Presenter receives `{ value, bind, meta, overrides, draft, mode }`:
`meta` from the backend Attr (label/help/info/copy resolved per discriminant/constraints/kind),
call-site `overrides` win, `draft` = the ENTIRE projected object — safe by construction
(unexposed fields don't exist on the type and never crossed the wire). **`requires` gate:**
presenter declares meta it needs ⇒ using it on a field lacking that meta = compile error
(per-field meta types from codegen) + dev-mode runtime backstop.

### Auth — LOCKED (2026-07-18): adapter in, verdict out

**Authentication is not AD's problem.** Contract = three sentences: something upstream turns
a request into `ctx`; the app defines `AppContext`; AD never looks inside it. Clerk = ~20
lines of app-side Hono middleware in front of the existing adapter — swap providers by
rewriting those lines, zero AD/controller changes (controllers consume `ctx.userId`, never
`clerk.*`). **Authorization is already built:** `scopeBy` (rows) · `expose` (read) ·
`permit(ctx, record)` (write) · `abilities`/`can` (shipped verdict) · role-in-model ban.
**Presenters never receive roles** — `abilities` IS the auth status (server-computed, drift-
proof); presenters are app components and may use the app's own auth context (Clerk hooks)
for *decoration* only (e.g. "upgrade to edit" copy). Enforcement from the verdict,
decoration from context, never the reverse. Docs task: "Auth" page under Controllers
(adapter recipe: Clerk / Auth.js / test-fake) → add to Phase 4.

### Error architecture — LOCKED (2026-07-18)

One recursive `ErrorBag` per FormSession: `{ fields, base, children }`. Sources funneled:
client `validate()` ∪ server 422 (deduped) · re-fielded invisible-field issues · transition
guard failures · 409 text. Nested routing by identity (`id` / `_key` echo), never index.
Uniform API at every depth: `handle.field.errors` / `bind.errors` / `handle.$errors.any|count`
/ array rollup `loan.assets.$errors` / `<BaseErrors/>`. Presenters are depth-blind.

## Design decisions (locked / to lock)

1. **Projection = `expose` (data availability), not permit (editability).** A validator
   reading `status` (view-only) + `amount` (editable) SHOULD run client-side — the client
   has the data. `deps ⊆ expose` is the fail-closed boundary. This *changes* today's
   permit∪include projection — deliberate, migrate in 1d.1.
2. **`permit` signature breaks to `(ctx, record)`.** Pre-1.0; RFC form is right; `ctrl`
   reachable via closure where needed. Changelog note.
3. **No TanStack Form — LOCKED (2026-07-18).** The form problem is two layers: a data
   layer (values, codecs, dirty, validation, submission diff) and an interaction layer
   (touched, error timing, debounce, races, array keys, submit lifecycle). Our Client IS
   the data layer — the heavy half, already built. Riding TanStack means two stores of the
   same truth with per-field sync adapters between them; every sync boundary breeds bugs,
   and programmatic draft writes are invisible to it. Instead: **`FormSession`** in
   `@active-drizzle/react` wraps a Client with `{ touched, submitAttempted, isSubmitting,
   per-field listener map }`; each `Present` subscribes to its own field via
   `useSyncExternalStore` (keystroke re-renders one field). Client stays pure/isomorphic.
   We ride TanStack's *test suite*, not its runtime — the conformance matrix below is the
   certainty contract.
4. **`edit` is never inferred; `view` may default.** (RFC hard wall #2 — keep.)
5. **Meta is semantic only** — no layout/section/grid/step info on fields. (RFC wall #6.)
6. **Role/identity in model predicates = codegen error.** Roles live on controllers, full stop.
7. **`Attr.state` assignment stays legal, transition legality enforced at save-validate** —
   keeps the Attr contract (assign anything, validate on save) instead of throwing on set.
8. **Meta is OPEN, not enumerated (2026-07-18).** Core keys AD understands + arbitrary
   app keys (`icon`, `regulatoryDisclosure`, …). Two rules: (a) statically extractable —
   literals/nested data/analyzable pure fns — or build error; (b) typed via declaration
   merging on `AttrMeta`, so custom keys flow into `FieldMeta`, presenter props, and
   `requires` gates with full checking.
9. **Commit policy (2026-07-18).** Presenters declare their natural commit moment at
   registration (discrete inputs → on change; continuous → blur/debounce); context decides
   what commit DOES: `<Form>` = stage into draft (batch submit) · `<Form autosave>` =
   single-field PATCH per commit · standalone field = autosave by definition (handle owns
   id + mutation, zero props). `bind.state: idle|staged|saving|saved|error` lets any
   presenter render spinners/checkmarks. Optimistic write + rollback-on-failure + version
   echo (409 → refetch + notify) live in FormSession, written once — presenters stay ~30
   lines. The magic toggle: `<loan.isPublished edit />` = flip → optimistic → PATCH →
   rollback on fail, all from policy, none from the presenter.
10. **Headless forever.** AD ships the socket (contract, registry, binding, test-kit,
    one reference presenter as adaptation template), never the bulbs. No shipped UI, no
    CSS story, no silent default presenters.
